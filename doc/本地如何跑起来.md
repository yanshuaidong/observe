# 本地如何跑起来

本文档包含：第一次启动、以及后续启动的步骤（确保功能可用：前端能请求后端、后端能连上数据库并完成 `strategy` CRUD）。

## 第一次启动

### 1. 准备 MySQL
1. 启动 MySQL 服务，并确保能从本机连上（需要填写 `DB_HOST` / `DB_PORT` 等）。
2. 创建数据库（建议至少执行一次；`backend/db_init.py` 和后端都会依赖数据库已存在）：
```sql
CREATE DATABASE IF NOT EXISTS ysdclaw DEFAULT CHARACTER SET utf8mb4;
```
如果你在 `.env` 里改了 `DB_NAME`，把 `ysdclaw` 换成你的值。

3. 建表策略：后端启动不再自动建表；你需要在启动后端之前运行一次 `cd backend && python db_init.py`（见下文）。

### 2. 配置环境变量
在项目根目录把 `.env.example` 复制成 `.env`，然后填写数据库配置：
填写：`DB_HOST`、`DB_PORT`（默认 `3306`）、`DB_USER`、`DB_PASSWORD`、`DB_NAME`（默认 `ysdclaw`）。

可选：
- `CORS_ALLOW_ORIGINS`：允许跨域来源（默认 `*`，逗号分隔）

后端会从项目根目录自动读取 `.env`。

### 3. 启动后端（FastAPI）
1. 安装依赖：
```bash
cd backend
pip install -r requirements.txt
```
2. 一次性建表（只需跑一次；如果以后改了表结构再重跑）：
```bash
python db_init.py
```

如果你已经在 `observe/backend` 目录下，则直接运行 `python db_init.py` 即可。

3. 启动：
```bash
uvicorn app.main:app --reload --port 8000
```

4. 健康检查：
打开 `http://127.0.0.1:8000/api/health`，应返回：
```json
{"ok": true}
```

### 4. 启动前端（Vue3 + Vite）
1. 安装依赖：
```bash
cd frontend
npm install
```

2. 启动：
```bash
npm run dev
```

3. 打开地址：Vite 启动后会在终端输出 URL（默认通常是 `http://127.0.0.1:5173`）。

前端默认请求后端：`http://127.0.0.1:8000`（也可通过 `VITE_API_BASE_URL` 覆盖）。

### 5. MVP 联调快速验证
1. 打开前端首页（`/`，通常会重定向到 `/strategies`）。
2. 点击“新建策略”，填写 `strategyid` 和 `content(JSON)`，保存后回到列表页能看到新数据。

## 后续启动

### 1. 确保数据库可用
1. 启动 MySQL 服务（或保证它仍在运行）。
2. 默认无需重复建表；只有当你修改了表结构/DDL，才需要重新运行 `python backend/db_init.py`。

### 2. 启动后端（直接启动即可）
```bash
cd backend
uvicorn app.main:app --reload --port 8000
```

### 3. 启动前端（直接启动即可）
```bash
cd frontend
npm run dev
```

只有当你更新了 `backend/requirements.txt` 或 `frontend/package.json` 后，才需要重新跑一次 `pip install` / `npm install`。

## 常见问题

1. 连接不上数据库
请检查：`.env` 里的 `DB_HOST/DB_PORT/DB_USER/DB_PASSWORD/DB_NAME` 是否正确，以及 MySQL 是否允许该用户从本机连接。

2. 前端请求失败 / 跨域问题
请检查：前端会请求 `VITE_API_BASE_URL`（默认 `http://127.0.0.1:8000`）；如果你遇到 CORS 问题，把 `CORS_ALLOW_ORIGINS` 设置为你的前端地址（例如 `http://127.0.0.1:5173`）。

3. 表未创建
请在 `backend` 目录下执行一次 `python db_init.py`，或手动按 `database/策略1.md` 里的 DDL 创建 `strategy` 表。
