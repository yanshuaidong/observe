## 后端接口文档（FastAPI）

### 基本信息

- **服务名**：`observe-backend`
- **API 前缀**：`/api`
- **数据格式**：请求/响应均为 `application/json`（除 `204 No Content`）

### 通用约定

#### 通用错误响应（FastAPI 默认）

当接口抛出业务错误时，后端使用 FastAPI 的 `HTTPException`，响应格式一般为：

```json
{
  "detail": "error message"
}
```

#### 状态码约定（本项目实际用到）

- **200**：成功（GET/PUT）
- **201**：创建成功（POST）
- **204**：删除成功且无响应体（DELETE）
- **400**：参数错误 / JSON 不可序列化 / 路径与 body 不一致
- **404**：资源不存在
- **409**：资源冲突（例如 `strategyid` 已存在）

### 1. Health

#### 1.1 健康检查

- **方法**：GET
- **路径**：`/api/health`
- **请求参数**：无
- **返回值**：

| 字段 | 类型 | 含义 |
|---|---|---|
| ok | boolean | 服务是否可用（固定返回 `true`） |

- **示例响应**：

```json
{
  "ok": true
}
```

### 2. Strategies（策略）

#### 数据模型说明

后端存储表（见 `backend/db_init.py`）：

| 字段 | 类型 | 含义 |
|---|---|---|
| id | BIGINT (AI) | 自增主键（按插入顺序） |
| strategyid | VARCHAR(64) | 策略唯一标识（业务主键） |
| content | JSON | 策略内容（任意 JSON） |

接口的核心数据结构：

- **StrategyCreateRequest**

| 字段 | 类型 | 是否必填 | 含义 |
|---|---|---:|---|
| strategyid | string | 是 | 策略唯一标识（最短 1 字符） |
| content | any(JSON) | 是 | 策略内容（必须可 JSON 序列化） |

- **StrategyResponse**

| 字段 | 类型 | 含义 |
|---|---|---|
| strategyid | string | 策略唯一标识 |
| content | any(JSON) | 策略内容 |

- **StrategyListResponse**

| 字段 | 类型 | 含义 |
|---|---|---|
| items | StrategyResponse[] | 策略列表（按 `id` 倒序，即最新在前） |

---

#### 2.1 获取策略列表

- **方法**：GET
- **路径**：`/api/strategies`
- **请求参数**：无
- **返回值**：`StrategyListResponse`

| 字段 | 类型 | 含义 |
|---|---|---|
| items | array | 策略数组 |

- **示例响应**：

```json
{
  "items": [
    {
      "strategyid": "s1",
      "content": {
        "name": "策略1",
        "enabled": true
      }
    }
  ]
}
```

- **可能错误**：通常无（除数据库连接/服务异常）

---

#### 2.2 创建策略

- **方法**：POST
- **路径**：`/api/strategies`
- **请求参数（JSON body）**：`StrategyCreateRequest`

| 参数 | 位置 | 类型 | 是否必填 | 含义 |
|---|---|---|---:|---|
| strategyid | body | string | 是 | 策略唯一标识（不可重复） |
| content | body | any(JSON) | 是 | 策略内容（必须可 JSON 序列化） |

- **返回值**：`StrategyResponse`
- **成功状态码**：`201`

- **示例请求**：

```json
{
  "strategyid": "s1",
  "content": {
    "name": "策略1",
    "enabled": true
  }
}
```

- **示例响应**：

```json
{
  "strategyid": "s1",
  "content": {
    "name": "策略1",
    "enabled": true
  }
}
```

- **可能错误**：

| 状态码 | 场景 | detail 含义 |
|---:|---|---|
| 400 | `content` 无法 JSON 序列化 | `content is not JSON-serializable: ...` |
| 409 | `strategyid` 已存在 | `strategyid already exists` |

---

#### 2.3 获取单个策略

- **方法**：GET
- **路径**：`/api/strategies/{strategyid}`
- **请求参数（Path）**

| 参数 | 位置 | 类型 | 是否必填 | 含义 |
|---|---|---|---:|---|
| strategyid | path | string | 是 | 策略唯一标识 |

- **返回值**：`StrategyResponse`

- **示例响应**：

```json
{
  "strategyid": "s1",
  "content": {
    "name": "策略1",
    "enabled": true
  }
}
```

- **可能错误**：

| 状态码 | 场景 | detail 含义 |
|---:|---|---|
| 404 | 未找到对应 `strategyid` | `strategy not found` |

---

#### 2.4 更新策略（全量覆盖 content）

- **方法**：PUT
- **路径**：`/api/strategies/{strategyid}`
- **请求参数**

**Path**

| 参数 | 位置 | 类型 | 是否必填 | 含义 |
|---|---|---|---:|---|
| strategyid | path | string | 是 | 需要更新的策略唯一标识 |

**JSON body（StrategyCreateRequest）**

| 参数 | 位置 | 类型 | 是否必填 | 含义 |
|---|---|---|---:|---|
| strategyid | body | string | 是 | 必须与路径中的 `{strategyid}` 完全一致 |
| content | body | any(JSON) | 是 | 新的策略内容（覆盖写入） |

- **返回值**：`StrategyResponse`

- **示例请求**：

```json
{
  "strategyid": "s1",
  "content": {
    "name": "策略1-更新版",
    "enabled": false
  }
}
```

- **示例响应**：

```json
{
  "strategyid": "s1",
  "content": {
    "name": "策略1-更新版",
    "enabled": false
  }
}
```

- **可能错误**：

| 状态码 | 场景 | detail 含义 |
|---:|---|---|
| 400 | 路径 `strategyid` 与 body 的 `strategyid` 不一致 | `strategyid mismatch between path and body` |
| 400 | `content` 无法 JSON 序列化 | `content is not JSON-serializable: ...` |
| 404 | 未找到对应 `strategyid` | `strategy not found` |

---

#### 2.5 删除策略

- **方法**：DELETE
- **路径**：`/api/strategies/{strategyid}`
- **请求参数（Path）**

| 参数 | 位置 | 类型 | 是否必填 | 含义 |
|---|---|---|---:|---|
| strategyid | path | string | 是 | 需要删除的策略唯一标识 |

- **返回值**：无（`204 No Content`）

- **示例响应**：无响应体

- **可能错误**：

| 状态码 | 场景 | detail 含义 |
|---:|---|---|
| 404 | 未找到对应 `strategyid` | `strategy not found` |
