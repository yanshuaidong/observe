# 前端页面设计

## 整体思路

核心场景：**每天收盘后，像填表一样快速记录 → 长期积累后回顾对比 → 认识自己**。
前端设计围绕这三个动作展开。

---

## 一、页面结构（共 5 个主要页面）

### 1. 仪表盘首页 `/dashboard`

**解决的问题**：打开工具第一眼就知道"今天该做什么"。

- 顶部：今天日期 + "今日记录进度"（比如"3 个策略中，已记录 1 个，还剩 2 个"）
- 中间：所有"观察中"的策略卡片，每张卡片显示策略名称、状态、最近一条记录的日期、累计记录条数。**已记录今天的**卡片变成绿色/打勾，**未记录的**保持醒目状态
- 核心按钮：**"开始今日记录"** — 点击后进入每日记录流程，自动遍历所有未记录的策略
- 底部：最近 7 天的记录日历热力图（哪天记了、哪天缺了，一目了然，培养记录习惯）

### 2. 策略管理页 `/strategies`（改进现有页面）

**解决的问题**：增删改查策略本身（不是每日记录）。

- 从表格改为**卡片列表**布局，每张卡片显示：
  - 策略名称、描述摘要
  - 状态标签（观察中 = 蓝色、暂停中 = 灰色、已结束 = 黑色）
  - 创建日期、累计记录数、最后记录日期
- 顶部：状态筛选 Tab（全部 / 观察中 / 暂停中 / 已结束）
- 新建策略时：不再是填写原始 JSON，而是一个**结构化表单**，填写策略元信息（名称、描述、状态）；策略的"想法字段模板"可以选择预设模板（目前有两套：放量增仓、大单跟随），也可以用 JSON 自定义
- 点击卡片 → 进入策略详情页

### 3. 策略详情页 `/strategies/:strategyid`（新页面，最重要）

**解决的问题**：看一个策略的全部历史，理解自己在这个策略上的行为模式。

分为三个 Tab：

**Tab A：时间线（核心视图）**
- 所有记录按时间**倒序**排列，用时间线组件展示
- 每条记录是一张卡片，概要显示：日期、品种名、方向、操作类型、情绪（用 emoji 或颜色图标）、一句话摘要
- 左侧或顶部有**情绪色带**：贪婪=红、恐惧=绿、冷静=蓝... 让你滑过去一眼就能看出情绪变化的节奏
- 点击展开卡片 → 看完整记录内容（技术面、基本面、消息面、情绪详情）
- 卡片上有"编辑"和"删除"按钮
- 这个视图的价值：你能像翻日记一样回顾，比如"那两天贪婪连着恐惧，原来每次都是追涨杀跌"

**Tab B：策略信息**
- 展示和编辑策略元信息（名称、描述、状态）
- 展示策略的"想法字段模板"定义（让你知道每次记录要填什么）

**Tab C：阶段性总结**
- 展示所有总结记录
- 支持新建周总结/月总结
- 结构化表单：观察天数、触发次数、操作次数、盈亏、经验教训、是否调整等

### 4. 每日记录页 `/record`（新页面，核心工作流页面）

**解决的问题**：每天收盘后，快速、舒服地填写记录。

**设计理念**：像"问卷/向导"一样引导你填写，而不是面对一大堆空白字段。

- **步骤一：选策略** — 列出所有"观察中"的策略，点选一个（如果从仪表盘"开始今日记录"进来，会自动排列需要记录的策略）
- **步骤二：品种与操作** — 结构化表单：品种名（下拉+自定义）、合约代码、方向（三选一按钮组：做多/做空/观望）、操作（按钮组：未操作/开仓/平仓...）、价格、数量、止损止盈、原因（文本框）、有没有后悔（开关）
- **步骤三：来源** — 不同策略不同字段：
  - 策略1（放量增仓）：信息来源（单选）、详细记录（文本）
  - 策略2（大单跟随）：国泰君安推荐日期、推荐理由、推荐方向
- **步骤四：分析面** — 根据策略类型动态展示：
  - 策略1：技术面（三个开关：放量？增仓？上涨？）+ 价格/支撑/阻力 + 基本面列表（动态增删行）+ 消息面列表
  - 策略2：大单数据（动态增删行，每行：日期、净买入量、变化方向）+ 大单信号判断（开关 + 选项）+ 消息面列表
- **步骤五：情绪面** — 这一步要做得有仪式感：
  - 情绪选择：用**大图标按钮**而不是下拉框（贪婪、恐惧、犹豫、冷静、兴奋、焦虑、无感），每个图标配对应颜色
  - 情绪强度：滑块 1-5
  - 市场情绪/跟随信心：同上
  - 情绪备注：给一个大文本框，鼓励写下当时的真实想法
- **步骤六：确认并保存** — 预览整条记录的摘要，确认后保存

**关键设计点**：
- 步骤之间用 Tab 或 Stepper 导航，可以前后跳转，不强制线性
- 所有字段都有合理默认值，必填项少（只有品种名和方向），其他全部可选
- 保存后自动跳转到下一个待记录的策略，或返回仪表盘

### 5. 回顾与对比页 `/review`（未来可加，先规划）

**解决的问题**：长期积累后，跨策略、跨时间维度对比分析。

- **日历视图**：按月展示，每天一个格子，显示当天记录了哪些策略、主要情绪是什么
- **情绪统计**：饼图/柱状图，展示一段时间内各种情绪的占比。比如发现"60% 的操作都是贪婪驱动"
- **策略对比**：两个策略并排，对比触发次数、操作次数、后悔比例、情绪分布
- **筛选器**：按日期范围、品种、情绪、操作类型筛选记录

## 二、整体布局

- **左侧导航栏**（收起时只显示图标）：仪表盘、策略管理、每日记录、回顾分析
- **顶栏**：应用名称（比如"策略观察台"）、当前日期
- **主内容区**：各页面内容
- 整体风格：**简洁克制**，Element Plus 默认主题即可。核心追求信息密度和填写效率

---

## 三、优先级（分阶段实施）

1. **第一步**：改造策略详情页（时间线视图）+ 每日记录页（结构化表单替代 JSON textarea）— 每天都要用的核心功能
2. **第二步**：仪表盘首页 — 让每天打开工具就知道该做什么
3. **第三步**：改造策略管理页（卡片化 + 新建表单）
4. **第四步**：回顾与对比页 — 等积累了足够多的数据再做

---

## 四、关键架构：策略类型 → 表单组件映射

因为策略1和策略2的"想法字段"结构差异很大（一个有技术面三开关，一个有大单数据表），前端需要一个**策略类型 → 表单组件的映射机制**：

- 在 `content` 里新增一个字段 `strategyType`（比如 `"volume_chase"` / `"large_order_follow"`）
- 前端根据 `strategyType` 动态加载对应的表单组件和展示组件
- 对于还没有专属表单的策略类型，fallback 到 JSON 编辑器（保持 MVP 的兜底能力）

这样以后想到新策略，只需要定义一个新的 `strategyType` 和对应的表单/展示组件即可。

## 五、路由汇总

| 路径 | 页面名称 | 说明 |
|------|----------|------|
| `/` | - | 重定向到 `/dashboard` |
| `/dashboard` | 仪表盘首页 | 今日记录进度、策略概览、热力图 |
| `/strategies` | 策略管理页 | 策略卡片列表、新建/删除策略 |
| `/strategies/:strategyid` | 策略详情页 | 时间线、策略信息、阶段性总结（三个 Tab） |
| `/record` | 每日记录页 | 步骤式表单，核心工作流 |
| `/record/:strategyid` | 每日记录页（指定策略） | 跳过选策略步骤，直接进入填写 |
| `/review` | 回顾与对比页 | 日历视图、情绪统计、策略对比（后期） |
